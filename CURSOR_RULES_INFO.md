# Cursor Rules - How They Work

## ✅ Да, правила будут использоваться Claude Code!

### Как это работает:

1. **Автоматическое чтение**: Cursor автоматически читает файл `.cursorrules` из корня проекта
2. **Применение ко всем агентам**: Правила применяются ко всем AI агентам в Cursor, включая:
   - Claude Code (основной агент)
   - Claude Sonnet
   - GPT-4
   - Другие доступные модели

3. **Текущая структура**:
   - `.cursorrules` → симлинк на `.vscode/.cursorrules`
   - Реальный файл: `.vscode/.cursorrules` (125 строк)
   - Правила iOS/WatchOS добавлены в строки 52-95

### Что включено в правила:

✅ **Agent Expertise** - декларация экспертизы в iOS/WatchOS  
✅ **Code Quality Requirements** - обязательная проверка компиляции  
✅ **Architectural Patterns** - следование best practices  
✅ **State Management & Lifecycle** - правильная работа при навигации  
✅ **Screen Sleep Prevention** - предотвращение засыпания экрана  
✅ **Testing Requirements** - требования к тестированию  

### Когда применяются правила:

- При работе с iOS/WatchOS кодом (Swift, SwiftUI)
- При создании новых файлов
- При рефакторинге существующего кода
- При исправлении багов
- При написании тестов

### Проверка:

Чтобы убедиться, что правила работают, можно:
1. Открыть любой Swift файл в Cursor
2. Попросить агента внести изменения
3. Агент должен автоматически следовать правилам из `.cursorrules`
